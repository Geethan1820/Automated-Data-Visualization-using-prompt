                             (GVS DataNova)
================================================================================
Last Updated: February 2026

1. ABSTRACT
--------------------------------------------------------------------------------
In the era of big data, extracting meaningful insights from raw datasets is
often complex and time-consuming, requiring expertise in data science tools.
"GVS DataNova" democratizes data analysis by bridging the gap between natural
language and technical execution. Users upload raw datasets (CSV/Excel) and
interact with them using simple English prompts (e.g., "Show me the sales
trend over time"). Using a robust Python/FastAPI backend and a React-based
interactive frontend, the system automatically cleans data, interprets user
intent, and instantly generates relevant, customizable visualizations without
writing a single line of code. The system also supports follow-up prompts,
context memory, color customization, chart export, and PDF reporting.

2. INTRODUCTION
--------------------------------------------------------------------------------
2.1 Problem Statement:
Traditional data analytics tools (Tableau, PowerBI, Python/R) have a steep
learning curve. Non-technical users struggle to visualize their data quickly.
Existing solutions often lack an intuitive, "conversational" interface for
immediate chart generation.

2.2 Objectives:
- To develop a web-based platform for easy data upload and automated
  preprocessing with a data quality score.
- To implement an intelligent NLP parsing engine that translates natural
  language prompts into analytical queries using fuzzy matching.
- To provide instant, interactive, and customizable visualizations based on
  user queries.
- To support context memory for follow-up prompts (e.g., "Change it to blue").
- To enable users to export charts as PNG/JPG and download full PDF reports.

2.3 Scope:
The system supports structured tabular data (CSV/Excel). It focuses on
descriptive analytics (What happened?) through visualization. It is designed
for students, business analysts, and researchers who need quick insight
extraction without coding.

3. SYSTEM ANALYSIS
--------------------------------------------------------------------------------
3.1 Existing System:
- Manual coding (Pandas, Matplotlib) requires programming skills.
- GUI-based tools (Excel) are cumbersome for large datasets and lack
  prompt-based automation.
- SaaS tools (Tableau Public) require training and often cost money.

3.2 Proposed System:
- Automated Data Cleaning: Handle missing values, type detection, and data
  quality scoring automatically.
- NLP-Driven with Fuzzy Matching: Users type what they want; the engine
  matches column names even with partial or misspelled input (using thefuzz).
- Context Memory: Follow-up prompts like "make it a line chart" or "use blue"
  are understood and applied without repeating the full prompt.
- Web-Accessible: No installation required; accessible via any modern browser.

3.3 Feasibility Study:
- Technical: Built using standard, well-documented, open-source technologies
  (React + Vite, Python FastAPI).
- Economic: Low cost; relies entirely on free and open-source libraries.
- Operational: Intuitive three-stage UI (Upload -> Preview -> Dashboard)
  minimizes training needs.

6. API ENDPOINTS
--------------------------------------------------------------------------------
  GET  /
    - Health check. Returns: {"message": "GVS DataNova Backend is Running"}

  POST /upload
    - Accepts: multipart/form-data (CSV or XLSX file)
    - Returns: file_id, filename, columns[], score, stats (rows, cols,
               missing), column_details[], preview (first 10 rows)

  POST /query
    - Body: { "prompt": "...", "file_id": "..." }
    - Returns: intent object (chart_type, x_column, y_column),
               chart_data[], summary, insights[], custom_color, error flag

7. MODULES DESCRIPTION
--------------------------------------------------------------------------------
7.1 Data Upload & Preprocessing Module (Upload.jsx + data_processor.py):
  - Accepts CSV and XLSX files via drag-and-drop or click.
  - Validates format, reads with Pandas, cleans missing values.
  - Calculates a "Data Quality Score" (0-100%) based on completeness.
  - Each upload generates a unique UUID-based file_id for session tracking.
  - Cleaned dataset is saved to /data/{file_id}_clean.csv server-side.

7.2 Data Preview Module (DataPreview.jsx + /upload endpoint):
  - Displays column names, data types, and missing value counts.
  - Shows a paginated/scrollable table of the first 10 rows.
  - Shows overall row/column stats and data quality score.
  - User clicks "Continue to Analysis" to enter the chat dashboard.

7.3 Prompt Parsing Engine / NLP Engine (nlp_engine.py):
  - Function: parse_prompt(prompt, columns, last_context)
  - Step 1: Fuzzy-match column names in the prompt using thefuzz
    (confidence threshold: 75%). Matches even partial/misspelled names.
  - Step 2: Detect color keywords (red, blue, green, etc.) for chart styling.
  - Step 3: Strict chart type detection (e.g., "pie chart", "bar graph").
  - Step 4: Intent-based chart detection from keyword mappings:
      "trend", "over time", "growth"       -> Line Chart
      "distribution", "frequency", "spread"-> Histogram
      "composition", "share", "percentage" -> Pie Chart
      "donut", "doughnut"                  -> Donut Chart
      "area", "cumulative"                 -> Area Chart
      "correlation", "vs", "relationship"  -> Scatter Plot
      "heatmap", "matrix", "intensity"     -> Heatmap
      "box", "outliers", "quartile"        -> Box Plot
      Default                              -> Bar Chart
  - Step 5: Context memory — if prompt is vague (e.g., "make it red"),
    reuse columns and chart type from the previous query of that session.
  - Step 6: Auto-assign X/Y columns; time-series columns are auto-detected
    from column name keywords (date, time, year, month, day, quarter).
  - Returns a structured intent object.

7.4 Visualization Module (ChartDisplay.jsx):
  - Renders the correct Recharts component based on the intent object.
  - Supported chart types: Bar, Line, Area, Pie, Donut, Scatter,
    Histogram, Box Plot, Heatmap (as a styled grid).
  - Applies custom_color from the session context.
  - Interactive: tooltips, legends, responsive containers.
  - Export: download as PNG/JPG using html2canvas.

7.5 Chat & Context Module (Chat.jsx + SESSION_CONTEXT in main.py):
  - Users interact through a chat-style interface.
  - Each prompt is sent to /query with the current file_id.
  - Server maintains SESSION_CONTEXT (per file_id): chart_type, x_column,
    y_column, found_columns, custom_color for follow-up aware responses.
  - Bot responses include reasoning, auto-generated insights (e.g., trend
    percentage change, highest value, correlation), and error suggestions.

7.6 Export & Reporting Module:
  - Individual chart export as PNG/JPG via html2canvas (client-side).
  - Full-session PDF report generation using jsPDF + html2canvas.

7.7 Theme Module (ThemeContext.jsx + Sidebar.jsx):
  - Global dark/light mode toggle exposed via React Context.
  - All components respond to the theme, styled with Tailwind dark: classes.

8. DATA FLOW
--------------------------------------------------------------------------------
  Upload Flow:
  [User] -> (Drag/Drop File) -> [Upload.jsx] -> POST /upload ->
  [data_processor.py: clean + score] -> [/data/ saved] ->
  [DataPreview.jsx: show stats + preview]

  Query Flow:
  [User] -> (Type Prompt) -> [Chat.jsx] -> POST /query ->
  [nlp_engine.py: parse_prompt()] -> (Query Pandas DataFrame) ->
  [Return JSON: intent + chart_data + insights] ->
  [ChartDisplay.jsx: Render Recharts] -> [User views chart]

  Follow-up Flow:
  [User] -> ("Change it to blue") -> POST /query ->
  [nlp_engine.py: check SESSION_CONTEXT] -> (Reuse columns/chart_type) ->
  [Return same chart with custom_color="blue"] -> [ChartDisplay.jsx updates]

11. CONCLUSION
--------------------------------------------------------------------------------
"GVS DataNova" successfully simplifies data analysis by combining a robust Python
FastAPI backend (with fuzzy NLP parsing and Pandas processing) with a reactive
React frontend (featuring Recharts visualizations and a chat-style interface).
The context memory system enables multi-turn conversations, allowing users to
iteratively refine their charts. It reduces time-to-insight significantly
compared to manual plotting or dashboard tools.
